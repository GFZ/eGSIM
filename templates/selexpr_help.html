A Ground Motion database (gmdb) is a set of seismic records stored in a tabular model as horizontal rows of where vertical columns represent each record field.

A selection expression on a gmdb is just a string containing a Python expression that will be evaluated for each record, returning matching rows only (i.e., rows for which the expression is true). The expression involves at least one column, a comparison operator (==, !=, <, <=, >, >=) and any value (or algebraic expression) consistent with the column type. E.g.:

pga <= 0.01

Several simple expressions can be wrapped in parenthesis and combined with logical operators (& for logical and, | for logical or, and ~ for negation), e.g.:

(pga != nan) & ((magnitude > 5) | ~(event_time <= "2016-08-30"))

The columns of a gmdb and their type are listed below. Please note that:

  • Strings and float columns store missing values as "" (empty string) and nan, respectively. This can be used to select records where the value of interest is not missing, e.g.: pgv != nan
  • String column values must be quoted. Date-time columns (e.g., event_time) must be typed as strings (thus quoted) in ISO format: YYYY-mm-dd, YYYY-mm-ddTHH:MM:SS, e.g.: event_time <= "2016-08-31"
