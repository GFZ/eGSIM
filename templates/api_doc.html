eGSIM API documentation

A Web Application Programming Interface (API) is an interface that allows communication between systems over the web using defined URLs and data formats. The communication is performed by a client (human or software code) that sends a Request to the API, i.e. the message specifying what action or data is desired, and receives a Response, i.e., the message sent back by the web service, usually containing the the requested data (e.g., a web page, an image, or any kind of bytes sequence).

What we just described is exactly what happens whenever you hit the keyboard or the mouse on this web site, as what you see here is actually another form of interface communicating with the web service, a Graphical user interface (GUI). Whereas a GUI is simpler and might suit many user needs, an API overcomes several GUI limitations by allowing communicating more programmatically and flexibly via software code.

In general, we recommend the usage of the eGSIM API through Python code and consult our online examples in Jupyter Notebook: therein, you can see how we access the API via a single main function that can also be used seamlessly in any Python module or, with slight modifications, in any programming language. However, for a more low-level approach, e.g., using command line tools like curl, or for Python users who want more configuration options in their code, here below a short documentation of the eGSIM API.
